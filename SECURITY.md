# 보안 안내

## API 키·비밀값 관리

- **API 키( Gemini, Firebase 등)는 코드나 문서에 절대 넣지 마세요.**
- 로컬 개발: **`.env`** 파일에만 넣고, 이 파일은 **Git에 커밋하지 마세요.** (`.gitignore`에 포함됨)
- 배포(GitHub Actions 등): **GitHub Secrets** 등 비밀 저장소에만 저장하고, 빌드 시 주입하세요.

## 커밋 전 확인

- `git status`로 `.env`가 올라가 있지 않은지 확인
- 새로 만든 `.env`는 절대 `git add` 하지 말 것
- 문서(.md)에 실제 API 키를 예시로 붙여넣지 말 것

## 유출 시 조치

- 키가 저장소에 올라갔다면: 해당 키를 즉시 **폐기/재발급**하고, 새 키만 `.env`·Secrets에 등록하세요.
